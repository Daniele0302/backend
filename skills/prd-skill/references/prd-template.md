# PRD Template (Standard)

## 1. Title
- Product / feature name
- Document owner
- Date / version

## 2. Summary
- One paragraph overview of the problem and proposed solution

## 3. Background & Problem
- Current state
- Pain points / opportunity
- Evidence (research, data, anecdotes)

## 4. Goals
- Business goals
- User goals

## 5. Non-Goals (Out of Scope)
- Explicit exclusions

## 6. Target Users & Personas
- Primary users
- Secondary users
- Key assumptions

## 7. User Stories / JTBD
- Top 5-10 user stories or jobs-to-be-done

## 8. Solution Overview
- High-level approach
- Key user journeys

## 9. Scope & MVP
- MVP features
- Phase 2 / future enhancements

## 10. Functional Requirements
Use IDs for traceability (e.g., FR-1, FR-2).

| ID | Requirement | Priority | Acceptance Criteria |
|---|---|---|---|
| FR-1 |  |  |  |

## 11. Non-Functional Requirements
Use IDs (e.g., NFR-1).

| ID | Requirement | Target |
|---|---|---|
| NFR-1 |  |  |

## 12. UX / UI Notes
- Wireframe links (if any)
- Interaction notes
- Accessibility considerations

## 13. Data & Analytics
- Events to track
- Key funnels
- Dashboards / reporting

## 14. Dependencies & Integrations
- Internal dependencies
- External services / APIs

## 15. Risks & Mitigations
- Top risks
- Mitigation plan

## 16. Rollout / Release Plan
- Beta, rollout stages
- Feature flags
- Support / training

## 17. Success Metrics
- Primary metric(s)
- Guardrail metrics
- Baseline and target

## 18. Open Questions
- Items to resolve

## 19. Appendix
- Glossary
- References
